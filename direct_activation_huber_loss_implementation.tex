\section{Mục tiêu thay đổi}

Branch \texttt{feat/depth-shortcut-output-distill} được thêm một option mới cho shortcut predictor loss.
Trước thay đổi, các shortcut losses dùng output predictor theo dạng direction + log-magnitude:
\[
L_{\text{pair}}
=
\lambda_{\text{dir}} L_{\text{dir}}
+
\lambda_{\text{mag}} L_{\text{mag}}.
\]
Trong đó \(L_{\text{mag}}\) là mean squared error trên log-magnitude delta đã chuẩn hoá.

Sau thay đổi, code hỗ trợ thêm mode mới:
\[
\texttt{--shortcut-loss-mode direction\_activation}.
\]
Mode này coi output đầu tiên của predictor là activation dự đoán. Direction loss vẫn lấy hướng từ output này, còn loss phụ thay magnitude loss bằng Huber loss giữa activation dự đoán và activation target:
\[
L_{\text{pair}}
=
\lambda_{\text{dir}} L_{\text{dir}}
+
\lambda_{\text{mag}} L_{\text{act}}.
\]
Ở đây \(L_{\text{act}}\) là Huber loss giữa \(Y_{a \rightarrow b}\) và \(Z_b\).

\section{CLI mới}

Hai argument được thêm vào \texttt{train.py}:

\begin{verbatim}
--shortcut-loss-mode
--shortcut-activation-huber-delta
\end{verbatim}

\texttt{--shortcut-loss-mode} có hai giá trị chính:

\begin{itemize}
  \item \texttt{direction\_magnitude}: mode mặc định, giữ nguyên behavior cũ.
  \item \texttt{direction\_activation}: mode mới, dùng direction loss cộng activation Huber loss.
\end{itemize}

\texttt{--direct-loss-mode} và \texttt{direction\_activation\_huber} vẫn được accept như alias cũ để không làm gãy command cũ, nhưng tên mới nên dùng là \texttt{--shortcut-loss-mode direction\_activation}.

\texttt{--shortcut-activation-huber-delta} là tham số \(\delta\) của Huber loss, mặc định bằng \(1.0\).
Code validate rằng giá trị này phải lớn hơn \(0\).

\section{Direct pair forward}

Với một pair \((a,b)\), hidden states từ backbone là:
\[
Z_a, Z_b \in \mathbb{R}^{B \times N \times D}.
\]

Target layer activation luôn được stop-gradient:
\[
Z_b^{\text{target}} = \operatorname{stopgrad}(Z_b).
\]

Source hidden phụ thuộc vào loại pair:
\[
\tilde{Z}_a =
\begin{cases}
Z_a, & \text{joint pair},\\
\operatorname{stopgrad}(Z_a), & \text{predictor-only pair}.
\end{cases}
\]

Input vào predictor được tạo bằng helper \texttt{build\_predictor\_source}:
\[
U_a = \frac{\tilde{Z}_a}{\|\tilde{Z}_a\|_2 + \epsilon},
\qquad
m_a = \log(\|\tilde{Z}_a\|_2 + \epsilon).
\]

Nếu \texttt{--no-shortcut-predictor-normalize-input} được dùng, predictor nhận raw hidden \(\tilde{Z}_a\) thay vì \(U_a\), nhưng \(m_a\) vẫn được tính từ norm của hidden.

Predictor trả về:
\[
(Y_{a \rightarrow b}, \Delta m_{a \rightarrow b})
=
P(\text{source}, a, b, t_{\text{emb}}, m_a).
\]

\section{Direction loss giữ nguyên}

Direction prediction vẫn lấy hướng từ output \(Y_{a \rightarrow b}\):
\[
\hat{U}_{a \rightarrow b}
=
\frac{Y_{a \rightarrow b}}{\|Y_{a \rightarrow b}\|_2 + \epsilon}.
\]

Target direction:
\[
U_b
=
\frac{Z_b^{\text{target}}}{\|Z_b^{\text{target}}\|_2 + \epsilon}.
\]

Direction loss:
\[
L_{\text{dir}}
=
1 -
\mathbb{E}_{B,N}
\left[
\sum_D
\hat{U}_{a \rightarrow b} \cdot U_b
\right].
\]

\section{Mode cũ: direction\_magnitude}

Mode mặc định \texttt{direction\_magnitude} giữ nguyên loss phụ cũ.
Target magnitude delta:
\[
\Delta m^{\text{target}}_{a \rightarrow b}
=
\operatorname{clip}
\left(
\frac{\log(\|Z_b\|_2 + \epsilon) - \log(\|Z_a\|_2 + \epsilon)}
{\texttt{shortcut\_mag\_scale}},
-1,
1
\right).
\]

Magnitude loss:
\[
L_{\text{mag}}
=
\frac{1}{2}
\mathbb{E}
\left[
\left(
\Delta m_{a \rightarrow b}
-
\operatorname{stopgrad}(\Delta m^{\text{target}}_{a \rightarrow b})
\right)^2
\right].
\]

Pair loss:
\[
L_{\text{pair}}
=
\lambda_{\text{dir}} L_{\text{dir}}
+
\lambda_{\text{mag}} L_{\text{mag}}.
\]

\section{Mode mới: direction\_activation\_huber}

Mode \texttt{direction\_activation} thay \(L_{\text{mag}}\) bằng normalized element-wise Huber loss trên activation.

Activation prediction được lấy thẳng từ output đầu tiên của predictor:
\[
\hat{Z}_b = Y_{a \rightarrow b}.
\]

Target activation:
\[
Z_b^{\text{target}} = \operatorname{stopgrad}(Z_b).
\]

Activation Huber loss:
\[
s_b
=
\sqrt{
\mathbb{E}_{N,D}
\left[
\left(Z_b^{\text{target}}\right)^2
\right]
+ \epsilon
}.
\]
Scale \(s_b\) được tính riêng cho từng sample trong batch và được stop-gradient.

\[
L_{\text{act}}
=
\mathbb{E}_{B,N,D}
\left[
\operatorname{Huber}_{\delta}
\left(
\frac{\hat{Z}_b - Z_b^{\text{target}}}{\operatorname{stopgrad}(s_b)}
\right)
\right],
\]
với \(\delta = \texttt{shortcut\_activation\_huber\_delta}\).

Pair loss trong mode mới:
\[
L_{\text{pair}}
=
\lambda_{\text{dir}} L_{\text{dir}}
+
\lambda_{\text{mag}} L_{\text{act}}.
\]

Điểm quan trọng là \texttt{--shortcut-lambda-mag} vẫn là weight của loss phụ.
Nó không còn nhân magnitude loss trong mode mới, mà nhân activation Huber loss.

\section{Bootstrap shortcut loss trong mode mới}

Trong mode cũ, bootstrap dùng composition theo direction và log-magnitude:
\[
Z_a \rightarrow Z_b \rightarrow Z_c
\quad \text{so với} \quad
Z_a \rightarrow Z_c \text{ của EMA predictor}.
\]
Direction bootstrap vẫn dùng cosine giữa hướng của output composition và hướng của EMA output.
Magnitude bootstrap dùng delta-magnitude cộng dồn.

Trong mode \texttt{direction\_activation}, magnitude head không còn được dùng cho bootstrap loss.
Output \(Y_{a \rightarrow b}\) được coi là activation dự đoán tại layer \(b\).
Predictor bước hai nhận activation này làm source:
\[
Y_{a \rightarrow b} \rightarrow Y_{a \rightarrow b \rightarrow c}.
\]
EMA predictor tạo target activation:
\[
Y^{\text{ema}}_{a \rightarrow c}.
\]
Loss direction bootstrap vẫn là cosine trên hướng của hai output này:
\[
L_{\text{boot-dir}}
=
1 -
\mathbb{E}_{B,N}
\left[
\left\langle
\operatorname{norm}(Y_{a \rightarrow b \rightarrow c}),
\operatorname{norm}(\operatorname{stopgrad}(Y^{\text{ema}}_{a \rightarrow c}))
\right\rangle
\right].
\]
Loss phụ bootstrap trong mode mới cũng dùng normalized element-wise Huber. Với:
\[
s^{\text{ema}}_{a \rightarrow c}
=
\sqrt{
\mathbb{E}_{N,D}
\left[
\left(Y^{\text{ema}}_{a \rightarrow c}\right)^2
\right]
+ \epsilon
}.
\]

Loss phụ bootstrap là:
\[
L_{\text{boot-act}}
=
\mathbb{E}
\left[
\operatorname{Huber}_{\delta}
\left(
\frac{
Y_{a \rightarrow b \rightarrow c}
-
\operatorname{stopgrad}(Y^{\text{ema}}_{a \rightarrow c})
}{
\operatorname{stopgrad}(s^{\text{ema}}_{a \rightarrow c})
}
\right)
\right].
\]
Loss này thay cho \texttt{loss\_boot\_mag} và vẫn được nhân bởi \texttt{--shortcut-lambda-boot-mag}.

\section{Output distillation và FID skip}

Trong mode \texttt{direction\_magnitude}, output distillation và FID skip reconstruct hidden bằng hướng dự đoán cộng magnitude dự đoán:
\[
\hat{Z}_b = \exp(\hat{m}_b)\operatorname{norm}(Y_{a \rightarrow b}).
\]

Trong mode \texttt{direction\_activation}, magnitude head bị bỏ qua ở các nhánh này.
Hidden synthetic/resume được lấy trực tiếp từ output predictor:
\[
\hat{Z}_b = Y_{a \rightarrow b}.
\]
Vì vậy validation FID/sFID skip 3-to-7 cũng dùng activation output trực tiếp khi \texttt{--shortcut-loss-mode direction\_activation}.

\section{Gradient routing}

Thay đổi này không đổi rule gradient của direct pairs:

\begin{itemize}
  \item Joint pair: source \(Z_a\) không detach, nên gradient từ \(L_{\text{dir}}\) và \(L_{\text{act}}\) có thể đi vào predictor và backbone qua \(Z_a\).
  \item Predictor-only pairs: source \(Z_a\) và target \(Z_b\) đều stop-gradient, nên loss chỉ update predictor.
  \item Target \(Z_b\) luôn stop-gradient trong direct loss.
\end{itemize}

Trong mode mới, \(Z_b^{\text{target}}\) vẫn là hằng số theo gradient:
\[
\frac{\partial Z_b^{\text{target}}}{\partial \theta_{\text{backbone}}} = 0.
\]

\section{Aggregation qua nhiều pairs}

Logic sample 2 hoặc 3 direct pairs không đổi.
Nếu dùng 3 pairs:
\[
L_{\text{direct}}
=
\frac{
L_{\text{joint}}
+ L_{\text{ponly},1}
+ L_{\text{ponly},2}
}{3}.
\]

Nếu dùng 2 pairs:
\[
L_{\text{direct}}
=
\frac{
L_{\text{joint}}
+ L_{\text{ponly},1}
}{2}.
\]

Mode mới chỉ thay định nghĩa loss phụ bên trong từng pair.
Sampling truncated-normal/gap2-biased không đổi.

\section{Logging}

Các metric mới được thêm:

\begin{verbatim}
train/shortcut_loss_mode
train/shortcut_activation_huber_delta
train/loss_boot_aux
train/loss_direct_aux_joint
train/loss_direct_aux_ponly_mean
train/direct_activation_target_rms_joint
train/direct_activation_pred_rms_joint
train/direct_activation_rel_residual_rms_joint
train/boot_activation_target_rms
train/boot_activation_pred_rms
train/boot_activation_rel_residual_rms
val/FID_skip_output_mode
\end{verbatim}

\texttt{train/shortcut\_loss\_mode} được encode:
\[
0 = \texttt{direction\_magnitude},
\qquad
1 = \texttt{direction\_activation}.
\]

\texttt{train/loss\_direct\_aux\_joint} và \texttt{train/loss\_direct\_aux\_ponly\_mean} là tên trung lập cho loss phụ.
Trong mode cũ, chúng tương ứng với magnitude loss.
Trong mode mới, chúng tương ứng với activation Huber loss.

\texttt{train/loss\_boot\_aux} là tên trung lập cho bootstrap auxiliary loss.
Trong mode cũ, nó bằng magnitude bootstrap loss.
Trong mode mới, nó bằng activation Huber bootstrap loss.

\texttt{val/FID\_skip\_output\_mode} được encode:
\[
0 = \texttt{direction\_magnitude},
\qquad
1 = \texttt{direction\_activation}.
\]

Các metric cũ \texttt{train/loss\_mag\_joint} và \texttt{train/loss\_mag\_ponly\_mean} vẫn được giữ để không phá logging hiện tại.
Trong mode mới, hai metric này thực chất đang chứa activation Huber loss.

\section{Ví dụ sử dụng}

Để bật shortcut direction--activation loss:

\begin{verbatim}
--shortcut-loss-mode direction_activation \
--shortcut-activation-huber-delta 1.0
\end{verbatim}

Nếu không truyền hai argument này, training vẫn dùng behavior cũ:

\begin{verbatim}
--shortcut-loss-mode direction_magnitude
\end{verbatim}
